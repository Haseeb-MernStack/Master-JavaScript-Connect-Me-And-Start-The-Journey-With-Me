Function Declaration
In JavaScript, a function declaration is one of the standard ways to define a function.
Key Features:
Hoisted: You can call the function before it is defined in the code because JavaScript hoists function declarations.
Named: The function has a name (greet in the example) that can be used to reference and call it.
Function Expression
A function expression in JavaScript is when a function is assigned to a variable.
Key Differences from Function Declaration:
Feature - Function Declaration - Function Expression
Hoisting - Yes (can call before) - No (must define before use)
Naming - Named - Can be anonymous or named
Usage - Often used for top-level - Common with callbacks or passing functions
Arrow Function
An arrow function in JavaScript is a shorter syntax for writing functions. It was introduced in ES6 and is especially useful for simple or inline functions.
Parameters vs Arguments
Parameters
  • These are placeholders in a function definition.
  • You define them when writing the function.
  • Think of them as variable names the function expects.
    		•
    Arguments
  • These are the actual values you pass to the function when calling it.
  • They replace the parameters during execution.
    		•
    Default Parameters
    Default parameters in JavaScript let you assign a default value to a function parameter in case no argument is provided when the function is called.
    You can use default values for:
  • Strings
  • Numbers
  • Booleans
  • Arrays
  • Objects
  • Even other functions
    		•
    Rest parameters (...args)
    Rest parameters in JavaScript allow a function to accept an indefinite number of arguments as an array. It uses the ... syntax.
